# GTPJ 创意树

这个文件只是中文入口页，不维护独立排名，避免同一信息出现两套版本。

当前创意树已清空。新的创意必须从可靠来源、明确用户想法或可复核观察重新登记。

正式文件：

- `INDEX.md`：人工阅读的总索引，按当前框架版本排序。
- `idea_tree.json`：机器可读的创意注册表。
- `ideas/IDEA-xxxx_slug/IDEA.md`：单个创意的来源、假设、版本评分、迁移说明。
- `queues/`：从总索引派生出来的当前工作队列。

核心规则：

- 每个模块创新必须先有 `IDEA-xxxx`。
- 每个创意必须记录来源：论文、自己的想法、观察、跨学科来源或混合来源。
- 排序不是只看全局分，而是看当前框架版本的分数，例如 `version_scores.v1.score`。
- 一个 v1 创意可以继续用于 v2，但必须新增 `version_scores.v2`，不能直接沿用 v1 分数。
- 不同版本代码接口不同，所以同一创意在不同版本里的权重可以完全不同。
- 来源不清楚的创意不能开 trial。
- 实验代码和日志不放在创意树里；真正开始实现后，证据放到 `experiments/module_trials/`。
