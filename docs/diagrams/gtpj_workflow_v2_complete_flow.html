2. Evidence State Machine：正式实验怎么升级

当前状态不是手写结论，而是由 transition log 推导出来。

hypothesis_ready已有可检验假设、来源、风险、最小验证信号。
interface_precheck_passed输入输出、shape、baseline-off、GZSL hard rules 预检通过。
smoke_passed小跑通过，无 shape、metric、split、label 错误。
single_run_valid一次正式 run 有 manifest/result/quality/artifact refs。
tune_promising调参有价值，但不是 confirmed baseline。
ablation_supported消融支持模块贡献，能解释结果来源。
min3_confirmed3 次 clean repeat，有 mean/min/max/all runs。
promotion_candidate方法状态稳定，硬门清楚，reviewer allow。
promoted进入正式版本账本或 tag 流程。
blocked硬门、证据或工具条件不满足。
rerun_required证据可疑或运行失败，需要重新跑。
stopped / rejected无收益、复现不稳、消融不支持或预算停止。
TRANSITIONS.jsonl = 权威历史 evidence_routing.yaml = 当前状态缓存 合法推进： propose -> check -> apply -> append transition -> derive current_state 非法推进： 直接手改 evidence_routing.yaml failed hard rule 后 advance/promote campaign RESULT_INDEX 自建正式 H/U/S/ZS

3. Agent 怎么合作：不是聊天协作，是 transition 协作

agent 的职责不是“大家一起讨论”，而是围绕一个 subject 的状态迁移分别提出、检查、应用。

角色 默认存活时间 能做什么 不能做什么 典型输出
Coordinator workflow/campaign scoped 路由任务、冻结计划、apply transition、写最终账本 不能跳过 checker，不能单独宣布证据成立 task card, DECISION_LOG, applied transition
Planner workstream scoped 拆候选、生成 workstream、提出 hypothesis/candidate transition 不能改代码，不能写正式结果 candidate list, workstream plan
Implementer task scoped 只改一个 trial/code path 不能并行改同一路径，不能决定 keep/promote code diff, implementation notes
Runner Monitor / Experiment Runner campaign or run scoped 管 GPU、队列、run command、失败隔离 不能解释结果，不能写 promotion 结论 batch status, events, Warehouse receipts
Log Analyst run scoped 解析 U/S/H/ZS、best epoch，propose single_run_valid/failed 不能检查 GZSL 语义，不能 apply transition metric parse, proposed transition
Interface Checker task scoped 检查 input/output、shape、baseline-off、GZSL hard rules 不能修代码，不能启动 Runner interface_check, rule_checks
Quality Checker task scoped by default 检查证据链、artifact 边界、not_checked、是否允许 transition 默认不 campaign 长期驻留，不决定最终账本 quality_check, allow/block
Result Analyst / Reviewer task or workstream scoped 比较结果、复核争议、提出 repeat/stop/promotion_candidate 不能把 best repeat 单独当 promotion 依据 comparison summary, review notes

4. Mixed Campaign：创新3 + 调参20 + 消融 怎么执行

Campaign 层级

Campaign

解析 owner requested_mix，建立 campaign ledger。

  • campaign_manifest
  • WORKSTREAMS
  • AGENT_LIFECYCLE

Workstreams

按实验类型拆开。

  • innovation
  • tune
  • ablation
  • confirmation

Tasks / Runs

每个 task 有 subject_id 和 evidence_state。

  • TRIAL
  • ATTEMPT
  • RUN

调度原则

不是： 直接排 3 + 20 + N 个 run 而是： innovation 先 hypothesis/interface/smoke tune 只跑 frozen baseline/config ablation 等候有效候选 top candidate 才 min3 confirmation promotion 只由 evidence gate 触发

Workstream agents

范围角色
campaign scopedCoordinator, Runner Monitor, Campaign Result Comparator
innovation workstreamSource Reader, Innovation Planner, Implementer, Interface Checker, Reviewer
tune workstreamTune Planner, Runner, Log Analyst, Result Analyst, Quality Checker
ablation workstreamAblation Planner, Interface Checker, Runner, Result Analyst, Quality Checker
confirmation workstreamRunner, Log Analyst, Result Analyst, Quality Checker
run scopedExperiment Runner, Log Parser, Warehouse Registrar
Campaign 目录不能成为第二事实源。它只保存 routing facts 和 result_ref。正式 H/U/S/ZS 仍然来自 result.yaml、quality_check.md、agent_summary.md 和 Warehouse。

5. Innovation：一篇论文到多个 Trial

创新不是按论文名字堆实验，而是按假设和工程绑定拆。

Paper

论文、来源、官方代码、Research 长笔记。

→

Claim / Mechanism

论文里的一个机制或主张。

→

Hypothesis

为什么它可能提升 GZSL。

→

Interpretation

完整复现、轻量移植、loss-only、fusion-only、routing-only。

Attachment Point

text prototype、PSE、ICSA、local branch、classifier 前后。

→

Trial

hypothesis + interpretation + attachment point + implementation contract。

→

Attempt

同一 Trial 内的参数、seed、epoch、权重、top-k 尝试。

→

Evidence State

interface_precheck、smoke、single_run、tune、ablation、min3。

新开 Trial：改变科学假设、论文机制理解、接入点、forward 路径、新 loss、eval 语义或 baseline-off contract。
继续 Attempt：只改参数、seed、epoch、batch size、loss weight、top-k、hidden size。

6. GZSL 硬规范：哪些东西不能动

评估边界

  • seen/unseen split 不变
  • class order 不变
  • label mapping 不变
  • metric semantics 不变

形状边界

  • logits = [B（图片/样本数量）, C（类别数量）]
  • C 是 protected evaluation class count
  • 文本/视觉特征 shape 必须声明

泄漏边界

  • 不能泄露 unseen label
  • 不能暗改 eval 口径
  • 不能绕过 baseline-off

不清楚时

  • blocked
  • rerun
  • reject
  • 不能 keep/best/promote/confirmed
