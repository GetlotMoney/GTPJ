从 CLIP 开始的模块输入输出流程图（v4）

1. Frozen CLIP

输入：image batch

输出：clip_features

[B（图片数量）, 577（CLS+patch）, 768（CLIP维度）]

拆成 cls_token + patches

2. Text Cache + PSE

输入：GPT descriptions

输出：all_text

[200（类别数）, 768（文本维度）]

PSE = CLIP-A-self，v4 outer_ratio=0.5

3. ICSA

输入：cls_token + all_text

输出：all_text_cond

[B（图片数量）, 200（类别数）, 768（文本维度）]

同时生成 base_logits

4. Patch Select + FGVD

输入：patches

输出：jepa_patch_z, jepa_memory

[B（图片数量）, 32（选中patch）, 512（公共维度）]

FGVD = FAE geometry memory

5. BVSA

输入：FGVD memory + text

输出：score_s2v, score_v2s, local_score

[B（图片数量）, 200（类别数）]

双向视觉-语义对齐

主输出：logits

logits_200 = base_logits + 0.1 * local_score

训练输出：logits = [B（图片数量）, 150（seen类别数）]

评估输出：logits = [B（图片数量）, 200（全部类别数）]

指标：GZSL U/S/H/ZS

辅助监督：SGMP / BMDD / CE

SGMP 读 jepa_selected_patches, jepa_patch_z, jepa_memory, all_text_cond, labels。

mask topk=8，target = masked patch_z mean.detach，context = kept jepa_memory mean。

BMDD 读 score_s2v 和 score_v2s；CE 读 seen logits。

SGMP 不是无输入模块：它是训练时的 auxiliary loss 分支

输入张量

jepa_selected_patches: [B, 32, 768]，用于按条件文本选 masked patch。

jepa_patch_z: [B, 32, 512]，masked target 来源，target 会 detach。

jepa_memory: [B, 32, 512]，keep tokens 的 FGVD memory，作为 context。

all_text_cond: [B, 200, 768]，正样本/负样本条件文本。

labels: [B]，全局 seen 类 id，用来取正类和构造负类。

SGMP / AG-JEPA legacy

_ag_jepa_loss()

mask topk=8

context + text_z -> predictor -> pred

输出 loss

loss_mpp / legacy loss_jepa: 让 pred 接近 detached masked target。

loss_neg / legacy loss_jepa_neg: 负类条件文本 margin 约束。

最后进入总损失：loss += lambda_mpp * loss_mpp + lambda_neg * loss_neg。

ICSA GTPJ.forward() / meta_net cls_token = [B（图片数量）, 768（CLIP维度）]；all_text = [200（类别数）, 768（文本维度）] all_text_cond = [B（图片数量）, 200（类别数）, 768（文本维度）]；base_logits = [B（图片数量）, 200（类别数）] 用图片 CLS 生成轻量条件偏移，只改 seen 类文本；提供条件文本和基础 cosine logits。

BVSA CrossModalTransformer.decoder_v2s / decoder_s2v jepa_memory = [B（图片数量）, 32（选中patch数）, 512（公共维度）]；all_text = [200（类别数）, 768（文本维度）] score_v2s, score_s2v, local_score = [B（图片数量）, 200（类别数）] 视觉到语义、语义到视觉双向交互，产生局部对齐分数。

SGMP GTPJ._ag_jepa_loss() jepa_selected_patches, jepa_patch_z, jepa_memory, all_text_cond, labels loss_mpp/loss_jepa 和 loss_neg/loss_jepa_neg 用条件文本选 masked patch，预测 detached target；负样本文本走 margin 约束。

当前 v4 框架链路

v1: initial formal baseline

Status: confirmed, H=73.93

  • Frozen CLIP image backbone.
  • GPT class text prototypes.
  • Original text adapter path.
  • Patch bottleneck and geometry-aware visual encoding.
  • Original bidirectional visual-text interaction and auxiliary losses.

v2: PSE enters

Status: owner_activated_unconfirmed, best_observed_H=74.29

  • Parent: v1.
  • Source: IDEA-0001 / TRIAL-001 / ATTEMPT-019.
  • Change: CLIP-A-self sentence-level prototype adapter.
  • Modern name: PSE.
  • Evaluation semantics unchanged; confirmation still pending.

v3: conditional memory framework

Status: owner_accepted_stochastic_unconfirmed, best_observed_H=74.27

  • Parent: v2.
  • Source: IDEA-0002 / TRIAL-002 / ATTEMPT-004.
  • Adds strict main-path FAE-memory JEPA context.
  • Modern names: FGVD, BVSA, ICSA, SGMP.
  • Conditional text enters the alignment path as all_text_cond.

v4: confirmed tuned config

Status: confirmed, confirmed_H=74.45

  • Parent: v3.
  • Source: local-v3-054 min3 server confirmation.
  • Config only: pse_outer_ratio=0.5.
  • Config only: local_weight=0.1.
  • No code or evaluation-contract change; not auto activated.
PSE Prototype Semantic Enhancement. Legacy names: Adapter / CLIP-A-self.
ICSA Instance-Conditioned Semantic Adaptation. Legacy name: meta_net conditional text path.
FGVD Fine-grained Geometric Visual Descriptor. Legacy name: FAE geometry memory path.
BVSA Bidirectional Visual-Semantic Alignment. Legacy name: CrossModalTransformer path.
SGMP Semantic-Guided Masked Prediction. Legacy name: AG-JEPA auxiliary training.

V4 的正式含义：它是当前确认过的正式版本。它不表示已经运行 activate-version v4，也不表示当前 runtime alias 自动切换。 后续 tune、ablation、confirmation 默认应以 config/versions/v4.yaml 为正式基准。
