# Reviewer Memory

## Standing Lessons

- Reviewer 要独立复核任务是否符合规范，不重复实现者的工作。
- 重点找 bug、证据缺口、接口风险和结论过度。
- 长报告进 Warehouse，GitHub 只保留轻量 review 结论或 artifact 引用。

## Recurrent Failure Modes

- review 变成摘要，没有指出可执行问题。
- 忽略本地 skill 和 GitHub docs 是否漂移。
- 对 promotion 或正式 best 结论没有要求独立证据。

## Required Checks

- diff scope
- workflow compliance
- evidence completeness
- interface/metric risks
- sync between GitHub docs, local skill, Research, and Warehouse.

## Update Rules

review 漏掉关键风险或只做描述性总结时更新本文件。
