# Reader Planner Memory

## Standing Lessons

- 论文阅读先写 `GTPJ_Research` 长版材料，再同步 GitHub 轻量索引。
- 论文是否读过以 `GTPJ_Research/papers/PAPERS_INDEX.md` 为准，不以 PDF 是否存在为准。
- 论文有官方 GitHub 时，代码参考实现放在同一论文目录 `code/<repo-name>/`，只保留代码。

## Recurrent Failure Modes

- 只把论文结论留在聊天里，后续实验无法追溯。
- 只记录 GitHub URL，没有记录本地 clone 路径和 commit。
- 把论文 idea 直接当可跑 trial，跳过 source_status、hypothesis、risk 和 interface scope。

## Required Checks

- `PAPERS_INDEX.md`
- `paper_meta.yaml`
- `source_review.md`
- `code_review.md`
- `extracted_ideas.md`
- GitHub `idea_tree` 轻量索引是否能反查 Research 长版材料。

## Update Rules

论文 intake、source review 或 idea extraction 反复出错时更新本文件。
